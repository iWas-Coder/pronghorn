\section{Language-dependant Tools}

A modo de investigación y recolecta de información, se han recogido un conjunto de herramientas de análisis de código estático para cada uno de los lenguajes más comunes y/o utilizados. No todas estas herramientas van a resultar útiles, y es posible que algunas de sus funcionalidades ya estén integradas en otras, por lo que se van a proceder a analizar para comprobar tanto su funcionalidad, efectividad, utilidad, como su compatibilidad con las otras herramientas.

\subsection{Shell}

\textbf{Shellcheck}: warnings y sugerencias para scripts en bash/sh.

\subsection{Binaries}

\textbf{STOKE}: optimizador estocástico y sintetizador de programas para el conjunto de instrucciones x86\-64 (aplicable a binarios ELF de 64-bits).\\
\textbf{BinSkim}: análisis estático que proporciona resultados de seguridad y corrección para binarios portables de Windows.\\
\textbf{Ghidra}: framework de software reverse engineering (SRE) para analizar binarios, con funciones como desensamblaje, ensamblaje, decompilación, creación de gráficos, scripting, etc.\\
\textbf{Manalyze}: análisis estático para binarios portables que realiza una evaluación primaria, recopilando señales que podrían indicar un comportamiento malicioso, y mostrando información que podría ayudar a un análisis manual posterior.

\subsection{C/C++}

\textbf{CBMC}: permite verificar los límites de los \textit{arrays} (desbordamientos de búfer), seguridad de los punteros a direcciones de memoria, las excepciones y las aserciones especificadas por el usuario.\\
\textbf{CMetrics}: mide el tamaño y la complejidad de los archivos C.\\
\textbf{cppcheck}: análisis estático de código C/C++.\\
\textbf{cqmetrics}: métricas de calidad para código C.\\
\textbf{CScout}: métricas de complejidad y calidad para código C.\\
\textbf{flawfinder}: encuentra posibles debilidades de seguridad.\\
\textbf{Goblint}: análisis estático de código C \textit{multi-threaded}, para detección de \textit{data races}, desbordamiento de búfer, desreferencias de puntero nulo.\\
\textbf{scan-build}: durante la compilación de un proyecto, a medida que se compilan los archivos fuente, también se analizan en conjunto. Una vez completada la compilación y enlazado, los resultados se presentan en HTML.\\
\textbf{splint}: análisis estático de código C en busca de vulnerabilidades de seguridad y errores de codificación.\\
\textbf{SVF}: análisis estático de dependencias para código C/C++.

\subsection{Go}

\textbf{aligncheck}: encuentra estructuras empaquetadas ineficientemente.\\
\textbf{bodyclose}: comprueba si el cuerpo de la respuesta HTTP está cerrado.\\
\textbf{deadcode}: encuentra código no utilizado.\\
\textbf{dingo-hunter}: encuentra \textit{deadlocks}.\\
\textbf{dogsled}: encuentra asignaciones/declaraciones con demasiados identificadores en blanco.\\
\textbf{dupl}: informa de código potencialmente duplicado.\\
\textbf{errcheck}: comprueba que se utilizan valores de retorno de error.\\
\textbf{errwrap}: ajusta y corrige errores con la nueva directiva de verbo '\textit{\%w}'.\\
\textbf{flen}: obtiene información sobre la longitud de las funciones en un paquete.\\
\textbf{go-consistent}: analiza código y ajusta su consistencia.\\
\textbf{gochecknoglobals}: comprueba que no hay valores globales presentes.\\
\textbf{goconst}: encuentra cadenas repetidas que podrían ser reemplazadas por una constante.\\
\textbf{gocyclo}: calcula complejidades ciclomáticas de funciones en el código fuente.\\
\textbf{goimports}: comprueba las importaciones de paquetes faltantes o sin referencia.\\
\textbf{gokart}: análisis de seguridad con un enfoque en minimizar los falsos positivos. Es capaz de rastrear la fuente de variables y argumentos de funciones para determinar si las fuentes de entrada son seguras.\\
\textbf{goreporter}: ejecuta simultáneamente diversos linters y normaliza su salida en un informe.\\
\textbf{ineffassign}: detecta asignaciones ineficaces.\\
\textbf{maligned}: detecta estructuras que requerirían menos memoria si sus campos estuviesen ordenados.\\
\textbf{misspell}: encuentra palabras en inglés comúnmente mal escritas.\\
\textbf{nakedret}: encuentra \textit{naked returns}.\\
\textbf{nargs}: encuentra argumentos no utilizados en declaraciones de funciones.\\
\textbf{prealloc}: encuentra declaraciones de sectores que potencialmente podrían preasignarse.\\
\textbf{safesql}: protección contra inyecciones SQL.\\
\textbf{structcheck}: encuentra campos de estructura no utilizados.\\
\textbf{structslop}: recomienda reorganizaciones de campo de estructura para proporcionar la máxima eficiencia de espacio/asignación.\\
\textbf{test}: muestra la ubicación de los errores de los tests del módulo '\textit{stdlib}'.\\
\textbf{unconvert}: detecta conversiones de tipos redundantes.\\
\textbf{unparam}: encuentra parámetros de función no utilizados.\\
\textbf{varcheck}: encuentra variables y constantes globales no utilizadas.\\
\textbf{wsl}: hace cumplir con reglas de estilo para usar líneas vacías en los lugares adecuados.

\subsection{JavaScript}

\textbf{es6-plato}: muestra la complejidad del código fuente (ES6).\\
\textbf{escomplex}: análisis de la complejidad de los árboles de sintaxis abstracta de JavaScript.\\
\textbf{flow}: verificador de tipo estático para JavaScript.\\
\textbf{hegel}: verificador de tipos estáticos para JavaScript con un sesgo en la inferencia de tipos y sistemas de tipos fuertes.

\subsection{Python}

\textbf{bandit}: encuentra problemas de seguridad comunes en el código de Python.\\
\textbf{cohesion}: mide la cohesión de las clases definidas. Las clases que presentan una mayor cohesión presentan una complejidad reducida de sus módulos, una mayor mantenibilidad y reutilización.\\
\textbf{Dlint}: fomenta las mejores prácticas al escribir código y ayuda a garantizar que el código sea seguro.\\
\textbf{pyflakes}: comprueba el código de Python para ver si hay errores.\\
\textbf{pylint}: comprueba el código de Python para ver si hay errores.\\
\textbf{pyroma}: califica qué tan bien cumple un proyecto de Python con las mejores prácticas del ecosistema de empaquetado, y enumera los problemas que podrían mejorarse.\\
\textbf{pytype}: analizador de tipo estático para código Python.\\
\textbf{radon}: calcula varias métricas del código fuente (complejidad ciclomática, SLOC (líneas de código fuente), líneas de comentarios, líneas en blanco, medidas de complejidad de Halstead, índice de mantenibilidad, etc.)\\
\textbf{vulture}: Encuentre clases, funciones y variables no utilizadas en el código.
